Game Engines

Tech: Unity (C#), Godot (gdscript), Phaser (JavaScript & TypeScript), CodeSkulptor (Python)

I have dabbled in a few engines over the years with most time spent in Unity by a wide margin. Lately I have come to appreciate not only the simplicity of Godot but also the coziness of gdscript. I have made many prototypes but have yet to release any hobby-project game. Unity (and thus C#) was what the University I attended taught in.

For a few years I worked on some projects with two 3D artist friends of mine. We worked on a multiplayer party game that combined baseball mechanics in a competitive arena knock-em-out, called Bounce Ball. After that prototype we attempted a co-op top-down Zelda-esque game with dungeons and an overworld, called Gold Greed, where the gimmick was that you could carry each other - adding verticality to the puzzles and traversal. We worked on more game ideas and the common theme in all of them was the local multiplayer feature. I was the only programmer on those projects and spent a lot of time working on things like:

  • Character Controllers: Some games had controllers that were physics driven, some using Unity CharacterController class, some custom using arcade-style controls
  • GamePad Input: Allowing multiple gamepad types and also mouse+keyboard support, mostly using custom written wrappers/middleware to handle buttons and joysticks, until Unity released their vastly improved InputPackage feature
  • Menus: Coding and setting up everything from simple grid-button menus to more designed menus, all navigatable and controllable with both gamepads and mouse+keyboard
  • Animation Controllers: Since the character controllers generally were in direct contact with the animation controllers I was the one putting them together. Bounce Ball had the most advanced of animation controllers I have worked on, using layers and custom animations for all cardinal directions and abilities, etc
  • Gameplay Mechanics: A 2D-limited physics driven ball in a 3D game, Event systems, Environment Interactables, 2D ship controls, 2D jetpack controls, and more
  • Developer Features: An FPS counter, a time manipulation tool, editor tools to allow non-coders to create content (e.g weapons as a ScriptableObject)

I also made a simple metronome in Unity using a 3rd party plugin that would allow for more accurate timers. This was a personal project for a friend that could not afford a hardware metronome and also wanted more features, like custom sounds, custom time signatures, and BPM with option to tap it.
